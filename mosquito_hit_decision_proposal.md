# 모기 제거 성공 여부 판단 문제 제안

## 1. 문제 재정의: 좌표 예측에서 제거 성공 판단으로

### 기존 문제

- 과거 400ms 동안 관측된 모기의 3D 좌표를 이용해 `+80ms` 시점의 정확한 `(x, y, z)` 좌표를 예측
- 즉, 모델의 목표는 미래 위치를 최대한 정확히 맞히는 회귀 문제

### 제안 문제

- 현실 세계의 최종 목적은 좌표 예측 자체가 아니라 **모기를 제거하는 것**
- 따라서 문제를 다음과 같이 재정의

```text
0ms 시점에서 발사 판단을 했을 때,
80ms 뒤 실제 모기를 맞힐 수 있는가?
```

### 시스템 가정

- 레이저 시스템을 "굵은 비비탄처럼 일정 반경을 가진 직선 발사체"로 단순화
- 발사체의 유효 반경 `R` 안에 모기가 들어오면 제거 성공으로 판단

```text
조준 위치와 실제 +80ms 위치 사이의 거리 <= R  -> 명중 성공
조준 위치와 실제 +80ms 위치 사이의 거리 > R   -> 명중 실패
```

결과적으로 기존의 3D 좌표 회귀 문제를 **명중 가능성 이진분류 문제**로 변환한다.

## 2. 새로운 Train 데이터셋 구성

### 사용 데이터

- `open/train/*.csv`
  - 각 샘플의 `-400ms`부터 `0ms`까지 11개 시점 좌표
- `open/train_labels.csv`
  - 각 샘플의 실제 `+80ms` 좌표
- `open/test/*.csv`
  - `+80ms` 정답이 없으므로 이번 제안에서는 사용하지 않음

### 라벨 생성 방식

`0ms` 시점에서 발사한다고 가정하고, 최근 움직임을 이용해 조준점을 계산한다.

```text
최근 속도 = (0ms 위치 - -40ms 위치) / 0.04초
조준 위치 = 0ms 위치 + 최근 속도 * 0.08초
```

그 후 실제 `+80ms` 위치와 조준 위치의 3D 거리 오차를 계산한다.

```text
error = distance(조준 위치, 실제 +80ms 위치)
```

유효 반경 `R`을 기준으로 새 라벨을 만든다.

```text
error <= R  -> hit = 1
error > R   -> hit = 0
```

### 새 데이터셋 예시

```csv
id,hit
TRAIN_00001,1
TRAIN_00002,0
TRAIN_00003,1
...
```

### 유효 반경 후보

- `R = 0.03m`: 3cm 이내 명중, 난이도 높음
- `R = 0.05m`: 5cm 이내 명중, 현실성과 난이도의 균형
- `R = 0.10m`: 10cm 이내 명중, 비교적 쉬운 문제

초기 실험에서는 `R = 0.05m`를 기준값으로 사용하고, 난이도 조절을 위해 여러 반경을 비교할 수 있다.

## 3. 학습 및 평가 설계

### 데이터 분할

기존 train 데이터 10,000개만 사용해 새 라벨을 생성한다.

```text
학습 데이터: 80% = 8,000개
평가 데이터: 20% = 2,000개
```

재현성을 위해 무작위 분할 시 random seed를 고정한다.

```text
random_seed = 42
```

### 모델 입력

기본 입력은 기존 11개 시점의 3D 좌표다.

```text
11개 시점 * 3축 = 33개 피처
```

추가로 다음 파생 피처를 사용할 수 있다.

- 최근 속도
- 평균 속도
- 최근 가속도
- 속도 크기
- 가속도 크기
- 궤적 길이
- 시작점과 끝점 사이 직선거리
- 궤적 길이 / 직선거리
- 방향 전환각

이 피처들은 모기의 움직임이 안정적인지, 갑자기 방향을 바꾸는지, 발사 판단이 위험한지를 설명하는 데 유용하다.

### 모델 후보

- Logistic Regression
- Random Forest
- XGBoost / LightGBM
- MLP
- GRU 또는 Transformer Encoder

짧은 시계열 데이터이므로, 초기 baseline은 좌표 및 물리 기반 피처를 활용한 `LightGBM` 계열 모델이 적합하다.

### 평가 지표

단순 accuracy보다 실제 방제 시스템 관점의 지표가 필요하다.

```text
명중 성공: +1점
발사 실패: -2점
발사 보류: 0점
```

또는 다음 분류 지표를 함께 사용한다.

- Precision: 발사한다고 판단한 경우 중 실제 명중 비율
- Recall: 실제 명중 가능한 경우 중 발사를 선택한 비율
- F1-score: Precision과 Recall의 균형
- AUROC: 명중 가능성 확률 예측 성능

### 핵심 기대 효과

- 기존 문제보다 현실적인 방제 의사결정 문제로 전환
- 정확한 좌표 예측이 어려운 상황에서도 발사 여부를 판단 가능
- 불확실한 궤적에서는 발사를 보류하는 전략 학습 가능
- AI 모델의 역할을 "좌표 예측기"에서 "실제 제거 성공 판단기"로 확장
