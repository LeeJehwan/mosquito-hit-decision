# 방법론 상세 설명 및 앙상블 성능 분석

> 이 문서는 **각 방법론이 무엇이고 왜 썼는지**, **앙상블이 기존 LightGBM baseline보다 점수가 높아진 이유**,
> 그리고 **판정 지연(latency)** 까지 한 곳에 정리한 통합 자료다. 차트는 `methodology_comparison.png` / `ranking_auroc.png`.
> 재현: `python experiments/run_methodology_comparison.py --radii 0.05 0.04 0.03 0.02 0.01`
> 우승 모델은 `--model-type ensemble`로 학습/추론에 바로 사용할 수 있다(§7).

---

## 0. 문제와 평가지표 한 줄 요약

- 조준점은 **고정 공식**(`aim = p(0) + v_recent·0.08`, 80ms 등속 외삽)으로 이미 정해져 있다.
- 모델의 역할은 *"이 고정 조준을 믿고 쏠지 말지"* 를 판단하는 **확률 P(hit)** 생성이다.
- 점수(대회식): **발사·명중 +1 / 발사·빗나감 −2 / 미발사 0** → `mean_hit_score`.
- 비용 비대칭(−2) 때문에 정답은 "많이 쏘기"가 아니라 **"맞을 게 확실한 것만 골라 쏘기"** 이고, 이는 곧
  **확률 랭킹 품질(AUROC) + threshold 최적화** 싸움이다.

## 1. 평가 프로토콜 (공정 비교)

단일 검증셋의 우연을 줄이기 위해 다음 절차로 모든 방법론을 동일하게 측정했다.

1. train 8,000개에 **StratifiedKFold(5) OOF** 확률 생성 (각 샘플은 자기가 빠진 fold 모델로만 예측 → 누수 없음).
2. OOF에서 `hit_score`를 최대화하는 **threshold 선택** (단일 valid split보다 저분산).
3. 전체 train으로 **재학습** 후 **test 2,000개**로 최종 점수 확정 (test 라벨은 평가에만 사용).
4. 보정 모델은 비용 구조에서 도출되는 **이론 threshold = 2/3** 도 함께 평가.

> 비용 기반 threshold 근거: 기대보상 = `1·P + (−2)·(1−P) = 3P − 2`. 발사가 이득이려면 `3P − 2 > 0 ⇔ P > 2/3`.
> 즉 **확률이 잘 보정되면 최적 threshold는 0.667 근처**가 된다.

## 2. 특징 세트: baseline(114) vs adv(+17)

- **baseline 114개**: 위치(33) + 속도(30) + 가속도(27) + 최근/평균 속도·가속도 + 속력·가속도·회전각 요약통계 + 경로형태(path_length, tortuosity 등). → "궤적 모양"을 폭넓게 기술.
- **adv 17개**([`src/features_advanced.py`](../src/features_advanced.py)): "고정 선형 조준이 빗나갈 위험"을 **직접 정량화**.
  - **외삽 백테스트(7개, 핵심)**: 라벨을 만든 그 등속 외삽을 **과거 구간에서 똑같이 재현**해 실제 오차를 측정. 예) `-120ms→-80ms` 속도로 `0ms`를 예측 → 실제 `0ms`와의 거리. 이 과거 오차가 미래 외삽 오차의 강력한 대리지표(반경과 같은 단위).
  - 등속 vs 등가속 조준 간극(2), 저크(3), 속도 일관성(4), 최근 회전각(1).

표에서 **`+adv`** 가 붙은 방법론 = baseline 114 + adv 17 (총 131개).

## 3. 테스트한 8개 방법론 상세

### (1) LightGBM (baseline) — 기준선
- **무엇**: Gradient Boosting Decision Tree. leaf-wise(best-first) 성장으로 깊고 비대칭인 트리를 만들어 미세한 feature 상호작용을 잡는다.
- **왜**: 원 프로젝트의 기본 모델이자 모든 개선을 비교할 **대조군**. 비선형 패턴에 강하다.
- **설정**: 114 feat, n_estimators=300, num_leaves=31, lr=0.05.

### (2) Logistic Regression + StandardScaler (baseline) — 선형 하한선
- **무엇**: 선형 분류기. 위치·속도·가속도의 **스케일 차이**가 크므로 `StandardScaler`로 표준화 후 학습.
- **왜**: "이 문제가 선형으로 얼마나 풀리나"를 보는 **하한선**. 가장 단순·빠르고 해석 가능.

### (3) Logistic Regression + adv — 특징 효과 분리용
- **무엇**: 같은 선형 모델에 **adv 특징만 추가**.
- **왜**: 모델은 그대로 두고 **특징의 순수 기여만** 떼어 보기 위함. (결과적으로 가장 중요한 인사이트를 줌 → §5.1)

### (4) HistGradientBoosting + adv — 다른 계열의 GBDT
- **무엇**: sklearn의 히스토그램 기반 GBDT. **level-wise(깊이 제한)** 성장 + 강한 L2 정규화로, LightGBM과 **다른 방식의 트리**를 만든다.
- **왜**: 같은 "부스팅 트리"라도 분할·정규화 방식이 달라 **LightGBM과 오류가 비상관**. 앙상블 다양성의 한 축.

### (5) MLP (신경망) + adv — 딥러닝 대표 + 최속 추론
- **무엇**: 피드포워드 신경망(은닉층 128–64, ReLU, early stopping). `StandardScaler` 후 학습.
- **왜**: 트리와 전혀 다른 **매끄러운(continuous) 결정경계**를 학습 → 오류 구조가 트리와 크게 다름. 심층신경망 수업 맥락의 대표 모델이며 **단일 추론이 가장 빠르다**(행렬곱 몇 번, 0.21ms).

### (6) LightGBM + adv — 강한 모델 + 좋은 특징
- **무엇**: (1)에 adv 특징 추가.
- **왜**: "강한 트리 모델에 물리 특징을 더하면?"의 직접 검증. 단일 모델 중 상위.

### (7) LightGBM + adv (calibrated) — 확률 보정 + 비용 threshold
- **무엇**: `CalibratedClassifierCV(LightGBM, method="isotonic", cv=5)`. 예측 확률을 **단조 보정**해 신뢰도를 높임.
- **왜**: 확률이 잘 보정되면 §1의 **이론 threshold(2/3)** 가 실제로 최적이 되고, **threshold 과적합을 방지**(§5.4). r=0.03에서 1위, 평균 2위.

### (8) Ensemble(LGBM + HistGBM + MLP) + adv — 최종 우승
- **무엇**: 세 모델의 **확률을 평균**하는 소프트 보팅(hard voting이 아니라 `predict_proba` 평균 → 신뢰도 보존).
- **왜**: 서로 **다른 inductive bias**(leaf-wise 트리 / level-wise 트리 / 매끄러운 신경망)를 가진 모델을 결합해 **오류를 상쇄**. 5개 반경 평균순위 **1.6위(전체 1위)**.

## 5. 핵심 분석 — 앙상블은 왜 baseline LightGBM보다 점수가 높은가

baseline LightGBM → Ensemble의 개선은 **두 가지 독립적 원인**으로 깔끔하게 분해된다:
**① 특징(adv) 효과** 와 **② 앙상블(모델 결합) 효과**.

### 5.1 개선의 분해

| 반경 | ① LGBM baseline | LGBM **+adv** | Ensemble **+adv** | 특징 효과 Δ | 앙상블 효과 Δ | **총 개선** |
|---|---|---|---|---|---|---|
| 0.05 | 0.881 | 0.876 | 0.885 | **−0.005** | +0.009 | +0.004 |
| 0.04 | 0.839 | 0.839 | 0.842 | −0.000 | +0.003 | +0.003 |
| 0.03 | 0.773 | 0.776 | 0.785 | +0.003 | +0.009 | +0.012 |
| 0.02 | 0.652 | 0.659 | 0.661 | +0.007 | +0.002 | +0.009 |
| 0.01 | 0.211 | 0.232 | 0.245 | **+0.021** | +0.013 | **+0.034** |

읽는 법:
- **특징 효과**는 반경이 작아질수록 커진다(r=0.01에서 +0.021). 어려운 문제일수록 "외삽이 빗나갈 위험"을 직접 잰 adv 특징이 결정적.
- **앙상블 효과**는 거의 모든 반경에서 추가로 +. 특히 r=0.05에서는 특징이 손해(−0.005)였는데 앙상블이 +0.009로 **뒤집어 복구**한다.

### 5.2 앙상블이 작동하는 원리 — 오류 비상관 → 분산 감소

확률 평균이 단일 모델보다 나은 이유는 **bias–variance 분해**로 설명된다. 세 모델은 **서로 다른 방식으로 틀린다**:
- **LightGBM**: leaf-wise 트리 → 특정 분할에 과적합되는 경향, 계단형 경계.
- **HistGradientBoosting**: level-wise + 강한 L2 → 더 매끈하지만 binning 기반의 다른 오류.
- **MLP**: 전역적이고 매끄러운 비선형 경계 → 트리가 계단으로 근사하는 영역을 연속적으로 표현.

각 모델의 오류가 **완전히 같은 방향이 아니므로**, 확률을 평균하면 개별 오류가 부분적으로 **상쇄**되어 분산이 줄고, 그 결과 **확률 랭킹(AUROC)이 안정적으로 향상**된다. 어려운 반경에서 이 효과가 뚜렷하다:

| AUROC | r=0.05 | r=0.04 | r=0.03 | r=0.02 | r=0.01 |
|---|---|---|---|---|---|
| LGBM baseline | 0.851 | 0.837 | 0.833 | 0.808 | 0.810 |
| **Ensemble + adv** | 0.866 | 0.851 | **0.859** | **0.829** | **0.833** |

### 5.3 왜 이 평가지표(−2 비용)에서 특히 점수로 이어지나

점수는 `fire iff P ≥ threshold` 결정에서 나온다. **AUROC가 높다 = hit와 miss를 경계 근처에서 더 잘 분리한다**는 뜻이고, 이는 곧
**"+1을 더 많이 담으면서 −2를 더 많이 걸러내는" threshold를 설정할 수 있다**는 의미다.
−2 페널티가 큰 작은 반경(r=0.01에서는 fire-all이 **음수 −0.263**, 즉 무작정 쏘면 손해)일수록 이 분리력 향상이 **점수로 직결**되어, 앙상블의 이득이 가장 컸다(+15.8% vs baseline LGBM).

### 5.4 확률 평활화 → threshold 안정성 (r=0.05의 역전 사례)

r=0.05에서 흥미로운 역전이 있다. **LGBM+adv는 AUROC가 0.875로 앙상블(0.866)보다 높은데도 점수는 더 낮다(0.876 < 0.885).**

| r=0.05 | threshold | precision | recall | 점수 |
|---|---|---|---|---|
| LightGBM + adv | **0.960** | 0.974 | 0.967 | 0.876 |
| Ensemble + adv | 0.815 | 0.971 | 0.983 | **0.885** |

원인: 단일 LGBM의 확률 분포가 0/1 근처로 뾰족해, OOF에서 고른 threshold가 **0.96으로 극단 과적합** → 너무 신중해져 **맞출 수 있던 +1을 놓침**(recall 0.967). 반면 앙상블은 세 모델 평균으로 **확률이 매끄러워져** threshold(0.815)가 일반화가 잘 되고 recall 0.983으로 **+1 기회를 더 담아** 점수가 높다.
→ 앙상블의 이득은 *랭킹*뿐 아니라 **확률 평활화로 인한 threshold 견고성**에서도 온다. (보정 모델이 점수 2위인 것도 같은 메커니즘.)

### 5.5 정리: 어려운 반경일수록 이득이 커진다

총 개선폭은 r=0.05의 +0.004에서 r=0.01의 **+0.034(상대 +15.8%)** 로 커진다.
- 쉬운 반경(큰 원): 양성률이 높아 fire-all만 해도 점수가 높음 → 개선 여지 작음.
- 어려운 반경(작은 원): 선별이 전부 → **adv 특징(위험 정량화) + 앙상블(분리력·견고성)** 이 가장 크게 기여.

## 6. 판정 지연(latency) — `experiments/benchmark_latency.py`

판정 시간 = 특징 추출(1건) + 모델 추론(1건). 특징 추출: baseline 2.70ms, advanced 3.86ms/건.

| 방법론 | 배치 2000(ms) | 처리량(건/s) | 모델 1건(ms) | 판정 총(ms) |
|---|---|---|---|---|
| LightGBM (baseline) | 33.8 | 59,200 | 1.84 | 4.54 |
| LogReg+Scaler (baseline) | 4.2 | 481,000 | 0.63 | 3.33 |
| LogReg+Scaler + adv | 4.6 | 438,000 | 0.51 | 4.37 |
| HistGradientBoosting + adv | 49.9 | 40,000 | 12.43 | 16.29 |
| MLP(신경망) + adv | 3.7 | 537,000 | 0.21 | 4.07 |
| LightGBM + adv | 10.2 | 195,000 | 0.57 | 4.43 |
| LightGBM + adv (calibrated) | 57.8 | 34,600 | 7.63 | 11.50 |
| 🏆 Ensemble(LGBM+HGB+MLP) + adv | 64.1 | 31,200 | 14.80 | 18.66 |

- **데이터가 40ms 간격, 예측 지평이 80ms** → 판정 지연이 이보다 작으면 실시간 가능. **모든 방법론이 충족**(최대 ensemble 18.7ms).
- 빠른 모델에선 **병목이 모델이 아니라 특징 추출(Python 루프, 2.7~3.9ms)**. 더 줄이려면 모델 교체보다 특징 추출 벡터화가 효과적.
- 정확도 1위 Ensemble은 가장 느림(약 4배)이지만 80ms 예산 안. 추론 최속 모델은 MLP(0.21ms).

## 7. 결론·권장 및 사용법

### 권장
- **점수 최우선** → **Ensemble(LGBM+HistGBM+MLP) + adv** (5개 반경 평균순위 1위). 근소차 2위는 **Calibrated LightGBM + adv**(더 단순·해석 가능, 비용 threshold 2/3 사용).
- **속도까지 고려** → **LightGBM + adv** (점수 4위지만 판정 4.4ms로 가장 빠른 축, §6).
- **한 줄 메시지**: *"앙상블이 baseline LightGBM을 이긴 이유는 두 가지 — (1) 외삽 위험을 직접 잰 물리 특징, (2) 서로 다르게 틀리는 모델들의 확률 평균이 랭킹(AUROC)을 높이고 threshold를 견고하게 만든 것. 문제가 어려울수록(반경↓) 그 효과가 커진다."*

### 우승 모델 사용법 (코드에 통합됨)
`--model-type ensemble`을 지정하면 학습·추론이 자동으로 advanced 특징을 만들고 소프트보팅 앙상블을 사용한다.
```bash
uv run python train.py --dataset-dir dataset --radius 0.05 \
  --model-type ensemble --output runs/ensemble_01
uv run python infer.py --dataset-dir dataset --radius 0.05 \
  --model-type ensemble --output runs/ensemble_01
```
구현: 모델 `src/model.py`(`SoftVotingEnsemble`, `create_ensemble`), 설정 `src/config.py`(`EnsembleConfig`),
특징 `src/features_advanced.py`. 산출물은 `hit_ensemble.pkl`로 저장된다.
